# Physics Difficulty Rater

V2 is an independent, text-only Qwen + LoRA multi-task classifier for junior
high-school physics question difficulty. It predicts five final business
levels (`送分题` to `压轴题`) and ten interpretable auxiliary features. The
`problem_structure` feature is a seven-tag multi-label prediction; the other
nine features are single-choice classifications. Physical domains are stored in
the non-training `feature_metadata.knowledge_domains` field, so they do not
compete with task-structure tags.

## Environment

Reuse the server's existing `QuRater` conda environment first.  This project
uses the same GPU stack (PyTorch, Transformers, PEFT, Accelerate) and Qwen
base model, but keeps code, outputs, and dependency documentation separate.
No local model/runtime is required on the Mac.

## Data contract

Do not train against the raw source `difficulty` field.  Train against the
final `difficulty_rating.difficulty_level` generated by the API prompt plus
V7 post-processing.  The raw 25k JSONL stays outside Git; derive curated JSONL
with the command below.

```bash
bash scripts/prepare_pilot_data.sh /path/to/physics_difficulty_tiku_rated_v2_results.jsonl
```

## Training

```bash
nohup bash scripts/server_run_train.sh \
  /path/to/Qwen3.5-4B \
  data/curated/teacher_v2.jsonl \
  outputs/v2_baseline \
  configs/v2_teacher_train.json \
  > train_v2.log 2>&1 &
```

Set `GPU_COUNT=2` before the command for two visible GPUs. Training does not
run evaluation automatically; model selection should use the held-out gold set
in a separate evaluation command once it is created.

```bash
python evaluate_difficulty.py \
  --model_path /path/to/Qwen3.5-4B \
  --checkpoint_dir outputs/v2_baseline \
  --eval_file data/gold/gold_v2.jsonl \
  --output_file outputs/v2_baseline/gold_metrics.json
```
