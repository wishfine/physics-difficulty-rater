V2 baseline run notes
=====================

Target: reproduce the API prompt + V7 post-process final label, not the source
`difficulty` field.  The raw 25k source has 5k records in each source bucket,
but those buckets are not accepted as supervision.

Before the first 4B training run:
1. Produce 3k--5k API+V7 teacher labels from the raw source and inspect the
   distribution/quality manifest.
2. Keep the current 133 labelled records only as a pipeline smoke-test set.
3. Reserve a manually adjudicated gold set (target: 500) before model selection.

The ten feature labels are auxiliary supervision and explanations. They must
not be used as a hand-written inference rule in V2. This frozen-18-compatible
release uses ten single-label CE heads, including the original nine-class
`problem_structure` field.

Planned runs: (1) 1024-token baseline, (2) 1536-token length ablation, and
(3) 2048-token length ablation. Use the same frozen split and calibration
protocol. Do not select a model using the gold test set.

V3 pairwise run notes
=====================

Objective: replace absolute API label imitation with a text-only QuRating-style
soft Bradley-Terry scorer. Local Qwen3-32B is the pair judge; Qwen3.5-4B LoRA is
the scalar student. Images are not uploaded.

Hard exclusion: source `difficulty` and derived `raw_difficulty` are known bad.
They are not candidate strata, prompts, targets, calibration labels, or metrics.
Only direct bidirectional Qwen3-32B votes create V3 supervision. The API+V7
teacher level may be used only to diversify candidate edges.

Planned sequence:
1. 2,000-question / 8,000-pair pilot, seed 42.
2. Inspect 100 pairs; require >=99% parse success and <10% high-position-bias quarantine.
3. Short student run and select by validation soft pairwise log loss.
4. Expand comparison graph only after the pilot passes.
5. Calibrate four frozen thresholds on a fixed reference population with the
   declared 20/20/30/20/10 distribution; later compare against trusted anchors.
